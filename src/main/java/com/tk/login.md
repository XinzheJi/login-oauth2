# 多租户登录认证系统架构文档

## 1. 系统概述

本系统是一个基于Spring Security和JWT的多租户登录认证系统，支持与外部系统A的OAuth2集成。系统采用了标准的三层架构（控制层、服务层、数据访问层），并结合了多租户设计模式，实现了用户认证、权限控制和租户隔离等功能。

## 2. 系统架构

### 2.1 整体架构

系统主要由以下几个核心模块组成：

- **认证模块**：负责用户登录认证、JWT令牌生成与验证
- **安全模块**：负责权限控制、安全过滤器链配置
- **租户模块**：负责多租户上下文管理、租户数据隔离
- **OAuth2集成模块**：负责与外部系统A的OAuth2认证交互

### 2.2 技术栈

- 基础框架：Spring Boot
- 安全框架：Spring Security
- 认证方式：JWT (JSON Web Token)
- 数据库交互：MyBatis
- 缓存：Redis
- 外部认证：OAuth2
- 日志：SLF4J + Logback

### 3.2 详细认证流程

1. **客户端请求**：客户端向`/api/auth/login`接口发送POST请求，包含用户名和密码
2. **认证控制器**：AuthController接收请求并调用AuthService.login()方法
3. **认证服务**：
   - 调用Spring Security的AuthenticationManager进行用户名密码验证
   - 验证通过后，获取用户详细信息
   - 获取用户关联的租户信息
   - 通过OAuth2TokenService获取系统A的访问令牌进行验证
   - 生成包含用户信息、权限和租户ID的JWT令牌
4. **响应结果**：返回包含JWT令牌、用户信息和租户ID的LoginResponse对象

## 4. 核心组件详解

### 4.1 认证模块

#### 4.1.1 AuthController

负责处理登录、登出和用户信息查询等API请求。

主要API:
- `/api/auth/login`：用户登录
- `/api/auth/logout`：用户登出
- `/api/auth/user/info`：获取当前用户信息

#### 4.1.2 AuthService

认证业务逻辑服务，负责实现用户登录认证流程。

主要功能:
- 调用Spring Security认证管理器进行用户名密码验证
- 获取用户详情和租户信息
- 获取系统A的OAuth2令牌进行验证
- 生成JWT令牌

#### 4.1.3 JwtTokenService

JWT令牌管理服务，负责JWT的生成、解析和验证。

主要功能:
- 生成包含用户信息、权限和租户ID的JWT令牌
- 验证JWT令牌的有效性
- 从JWT令牌中提取用户名、权限和租户ID等信息

### 4.2 安全模块

#### 4.2.1 SecurityConfig

Spring Security的配置类，配置安全过滤器链、认证管理器等。

主要配置:
- 禁用CSRF防护
- 配置无状态会话管理
- 配置URL访问权限（登录接口允许匿名访问，其他接口需要认证）
- 注册JWT认证过滤器和租户过滤器

#### 4.2.2 JwtAuthenticationFilter

JWT认证过滤器，负责从请求头中提取和验证JWT令牌。

处理流程:
1. 从请求头中提取JWT令牌
2. 验证JWT令牌的有效性
3. 从JWT令牌中提取用户信息和权限
4. 创建认证令牌并设置到安全上下文

#### 4.2.3 UserDetailsServiceImpl

实现Spring Security的UserDetailsService接口，负责加载用户信息和权限。

主要功能:
- 根据用户名和租户ID查询用户信息
- 加载用户的角色和权限信息
- 构建Spring Security的UserDetails对象

### 4.3 多租户模块

#### 4.3.1 TenantFilter

租户过滤器，负责从JWT令牌中提取租户ID并设置到租户上下文中。

处理流程:
1. 从请求头中提取JWT令牌
2. 从JWT令牌中提取租户ID
3. 将租户ID设置到TenantContext中
4. 请求处理完成后清除租户上下文

#### 4.3.2 TenantContext

租户上下文，基于ThreadLocal实现，用于在当前线程中存储租户ID。

主要功能:
- 设置当前租户ID
- 获取当前租户ID
- 清除当前租户ID

#### 4.3.3 TenantService

租户服务，负责租户的查询和管理。

主要功能:
- 根据业务租户ID查询租户信息
- 根据租户主键ID查询业务租户ID

### 4.4 OAuth2集成模块

#### 4.4.1 OAuth2TokenService

OAuth2令牌服务，负责获取和管理系统A的访问令牌。

主要功能:
- 获取系统A的OAuth2访问令牌
- 缓存OAuth2令牌到Redis
- 清除OAuth2令牌缓存

## 5. 权限控制

系统实现了基于RBAC（基于角色的访问控制）的权限模型：

- **用户-角色-权限**：用户关联角色，角色关联权限
- **方法级权限控制**：通过Spring Security的`@PreAuthorize`注解和自定义`@RequirePermission`注解
- **URL级权限控制**：通过PermissionInterceptor拦截器实现

## 6. 安全措施

- **密码加密**：使用BCrypt加密算法
- **JWT签名**：使用HMAC-SHA256算法进行签名
- **令牌过期**：JWT令牌设置有效期
- **请求过滤**：通过过滤器链进行安全检查
- **权限校验**：请求级和方法级的权限控制
- **租户隔离**：基于TenantContext实现数据隔离

## 7. 数据模型

核心数据实体：
- **User**：用户信息
- **Role**：角色信息
- **Permission**：权限信息
- **Tenant**：租户信息

## 8. 系统集成

### 8.1 与系统A的集成

系统通过OAuth2客户端凭证方式与系统A进行集成：
- 获取系统A的访问令牌
- 在用户登录时验证系统A的访问权限
- 缓存系统A的访问令牌到Redis中

## 9. 部署要求

- JDK 17或更高版本
- Redis服务器
- MySQL数据库
- 与系统A的网络连接

## 10. 可扩展性

- **多租户扩展**：支持添加新的租户和租户隔离策略
- **权限扩展**：支持动态添加和管理权限
- **认证方式扩展**：可集成更多认证方式（如LDAP、社交登录等）
